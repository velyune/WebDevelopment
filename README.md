# WebDevelopment

## Команды

- `docker-compose up` – запустить PostgreSQL
- `pnpm nx serve api` – запустить back-end
- `pnpm nx serve web` – запустить front-end

- `pnpm nx prisma:status` - проверяет текущее состояние базы данных и выводит информацию о уже примененных и ожидающих выполнения миграциях.
- `pnpm nx prisma:migrate` - создает новые миграции на основе изменений в Prisma-схеме и автоматически применяет их к базе данных (используется только в среде разработки).
- `pnpm nx prisma:studio` - запускает локальный веб-интерфейс (Prisma Studio) для удобного визуального просмотра и редактирования данных в базе.
- `pnpm nx prisma:deploy` - применяет все ранее созданные, но еще не выполненные миграции к базе данных (предназначено для production-окружения и CI/CD пайплайнов).
- `pnpm nx prisma:migrate` - обновляет Prisma Client на основе текущей схемы, генерируя актуальные TypeScript-типы для безопасного взаимодействия с базой данных из кода.
- `pnpm nx prisma:migrate` - автоматически заполняет базу данных начальными или тестовыми данными в соответствии с заранее настроенным seed-скриптом.
